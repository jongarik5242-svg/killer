# 🛡️ killer — AI Blocker Suite

Набір інструментів для моніторингу та блокування використання AI-сайтів у браузері. Призначений для контролю за учнями під час навчального процесу.

---

## 📁 Файли проєкту

| Файл | Опис |
|------|------|
| `ast.py` | Основний блокер: стежить за активним вікном і закриває браузер при виявленні AI-сайту |
| `check_1.py` | Аналізатор історії Chrome/Edge: витягує та зберігає записи про відвідування AI-сайтів |
| `newscript.py` | Звуковий сповіщувач: відтворює сигнал при виявленні AI-сайту |
| `shlyapa.py` | Фотофіксатор: робить знімок з вебкамери при відкритті AI-сайту |
| `pips.py` | Утиліта автовстановлення залежностей з `requirements.txt` |
| `requirements.txt` | Список Python-залежностей |
| `start_tracker.bat` | Bat-файл для запуску `ast.py` через Thonny pythonw |

---

## ⚙️ Вимоги

- Windows 10 / 11
- Python 3.8+
- Thonny (або будь-який інший Python-інтерпретатор)

### Залежності

```
pywin32
psutil
opencv-python   # для shlyapa.py
pycaw           # для newscript.py
```

Встановити всі залежності одразу:

```bash
python pips.py
```

Або вручну:

```bash
pip install pywin32 psutil opencv-python pycaw
```

## 🚀 Запуск

### Основний блокер (ast.py)

Стежить за заголовком активного вікна. Якщо виявляє AI-сайт — миттєво завершує процес браузера.

```bash
python ast.py
```

Або через bat-файл:

```
start_tracker.bat
```

> ⚠️ У `start_tracker.bat` прописано конкретний шлях до Thonny і робочої папки — відредагуй під свою машину.

### Аналізатор історії (check_1.py)

Читає файли `History` з Chrome та Edge, шукає відвідування AI-сайтів і зберігає результат у `visits_simple.txt`.

```bash
python check_1.py
```

Або з вказанням конкретного файлу History:

```bash
python check_1.py "C:\шлях\до\History"
```

**Приклад рядка у вихідному файлі:**
```
3 березня Понеділок — 14:22:01 — ChatGPT — Клас: 10-м3 — Профіль: Default
```

### Звуковий сповіщувач (newscript.py)

Відтворює звуковий сигнал при відкритті AI-сайту.

```bash
python newscript.py
```

### Фотофіксатор (shlyapa.py)

При виявленні AI-сайту активує вебкамеру і зберігає фото у поточну папку.

```bash
python shlyapa.py
```

Фото зберігаються у форматі: `ChatGPT_photo_2025-03-03_14-22-01.jpg`

## 🔍 Список блокованих сайтів

За замовчуванням відстежуються такі ключові слова в заголовку вікна:

```
chatgpt · openai · gemini · bard · claude · deepseek · grok · copilot
perplexity · midjourney · anthropic · characterai · mistral · ...
```

### Додати новий сайт

У файлі `ast.py` знайди рядок:

```python
STOP_WORDS = ["chatgpt", "openai", "gemini", ...]
```

І додай нове ключове слово, наприклад:

```python
STOP_WORDS = ["chatgpt", "openai", "gemini", ..., "perplexity"]
```

Те ж саме стосується `newscript.py` (список `AI_SITES`) та `shlyapa.py` (словник `ai_sites`).

## 📋 Формат виводу check_1.py

Результат записується у `visits_simple.txt`. Програма автоматично:

- Визначає сервіс (ChatGPT, Gemini, Claude тощо)
- Намагається розпізнати клас учня з URL або заголовку сторінки (формат: `10-м`, `9-ті` тощо)
- Якщо клас не знайдено — підставляє `Невідомий`
- Виводить дату у форматі з українськими назвами місяців та днів тижня

## 🔧 Відомі обмеження

- Всі скрипти працюють **тільки на Windows** (використовують `win32gui`, `winsound`, `pywin32`)
- `shlyapa.py` потребує підключеної вебкамери
- `newscript.py` потребує `pycaw` і може не працювати без аудіопристрою
- `start_tracker.bat` містить хардкодований шлях до Thonny — потрібно змінити під свою конфігурацію
- `check_1.py` копіює файл `History` тимчасово, оскільки Chrome блокує прямий доступ до відкритої БД
